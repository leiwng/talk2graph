# talk2graph — 技术实现方案

## 核心理念

**代码即状态，渲染是副作用。**

```
用户描述 → LLM 生成 TikZ 代码 → 浏览器渲染 → 精确数学图形
用户修改 → LLM 看到旧代码 + 修改指令 → 局部编辑 → 重新渲染
```

不在像素空间做修改（DALL-E/Midjourney 做不到一致性），而在代码空间做修改——代码没改的部分，渲染结果丝毫无差。

## 技术选型

| 层 | 技术 | 原因 |
|---|---|---|
| 图形语言 | TikZ / PGFPlots | LaTeX 生态的数学图形标准，LLM 训练数据量最大（arXiv 几十万篇论文） |
| LLM | OpenAI 兼容 API | 任意支持 Chat Completions 的模型均可接入 |
| 渲染 | tikzjax WASM | 浏览器端将 TikZ 源码直接编译为 SVG，无需后端 |
| 前端 | 单文件 HTML | 零依赖部署，任何静态托管都能跑 |
| 部署 | 腾讯云 COS 静态托管 | 国内 CDN 加速，HTTPS 自带证书 |

## 架构

```
┌──────────────────────────────────────┐
│             用户界面                  │
│   左侧：对话区   │   右侧：图形预览    │
│   输入框 + 历史  │   SVG 实时渲染     │
├──────────────────────────────────────┤
│           LLM 对话层                  │
│   System Prompt + 对话历史           │
│   + 当前 TikZ 源码（多轮修改上下文）   │
├──────────────────────────────────────┤
│         tikzjax 渲染层               │
│   TikZ 源码 → WASM 编译 → SVG        │
├──────────────────────────────────────┤
│          图形状态管理                 │
│   当前 TikZ 源码 + 历史栈 + 撤销       │
└──────────────────────────────────────┘
```

## System Prompt 设计

固定注入到每轮 LLM 调用，核心约束：

1. 所有输出必须是完整的 TikZ 文档（含 `\documentclass` ... `\end{document}`）
2. 标注文字用中文
3. 几何图形的长度、角度按用户要求精确设定
4. 函数图像用 `pgfplots` 的 `\addplot`
5. 辅助线用 `dashed` + `gray`
6. 修改时只改被要求的部分，其余保持原样

## 多轮修改实现

第一轮：System Prompt + 用户描述 → TikZ 代码

后续轮次：

```
System Prompt
+ 上一轮的完整 TikZ 源码（注入到 user message 中）
+ 用户的新修改要求
→ LLM 在旧代码上做增量编辑 → 新 TikZ 代码
```

关键技术点：LLM 看到的是完整的旧代码，不是"上次生成的图片"。代码中没被修改的 `\draw`、`\node`、坐标值，渲染结果 100% 一致。

## 部署

- 托管：腾讯云 COS 静态网站（广州节点）
- 文件：单个 `index.html`（约 16KB）
- tikzjax：通过 jsdelivr CDN 加载（约 5MB WASM 运行时）
- HTTPS：COS 自带证书
- 更新：`python3 deploy_cos.py` 一行命令
