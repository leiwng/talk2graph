# talk2graph — 技术实现方案

## 核心理念

**代码即状态，渲染是副作用。**

```
用户描述 → 内置模板 / LLM 生成 TikZ + SVG → 浏览器预览 → 精确数学图形
用户修改 → 当前 TikZ 源码 + 修改指令 → 局部编辑 → 重新渲染
```

不在像素空间做修改（DALL-E/Midjourney 做不到一致性），而在代码空间做修改——代码没改的部分，渲染结果丝毫无差。

## 技术选型

| 层 | 技术 | 原因 |
|---|---|---|
| 图形语言 | TikZ / PGFPlots | LaTeX 生态的数学图形标准，LLM 训练数据量最大（arXiv 几十万篇论文） |
| 模板生成器 | JavaScript 规则 + 数学参数 | 高频 K12 图形需要确定性、可测试的精确输出 |
| LLM | 服务端 OpenAI 兼容 API | 任意支持 Chat Completions 的模型均可接入，且不暴露浏览器 API Key |
| 渲染 | 内置 SVG + 服务端 node-tikzjax 兜底 + TikZ 导出 | 中文标注稳定预览；模板外图形可由服务端尝试渲染；TikZ 源码仍可下载用于 LaTeX |
| 前端 | 静态 HTML + JS | 静态页可直接跑内置模板，保留快速 MVP 路径 |
| 服务端 | Node.js HTTP 服务 | 提供 `/api/generate`、静态资源和模型代理 |
| 部署 | Node 服务优先，COS 静态演示可选 | 真实产品需要隐藏模型密钥；静态托管只适合模板演示 |

```
└──────────────────────────────────────┘
```

## 生成策略

优先使用内置模板覆盖产品需求中的确定性场景：

- 任意两条直角边的直角三角形并标注边长（自动计算斜边）
- 直角三角形内切圆（自动计算内切圆半径和圆心）
- 等边三角形、正方形、单圆半径/圆心、底边+腰长等腰三角形
- 单独的一次函数、二次函数、正弦函数图像
- `sin(x)` 和 `cos(x)` 在 `-2π` 到 `2π` 的图像
- 参数方程椭圆 `x=3cos(t), y=2sin(t)`
- 圆内接正多边形并标注圆心（支持中文边数）
- 内切圆半径可指定的等腰三角形
- 底角 30°、腰长 5cm 的等腰三角形
- `y=x²` 和 `y=2x+1` 的交点阴影区域
- 长方体斜截面图
- 班级成绩条形统计图、通用标签+数值柱状图
- 去掉边长标注、添加斜边上的高、移动直角标记、标出角 A/B/C、修改坐标范围、添加/虚线化辅助线

模板匹配遵循“精确优先，宁缺毋错”：复杂请求如果尚未实现（例如相交圆、坐标轴圆、分段函数、抛物线与圆交点），不会用单圆或单函数模板降级冒充，而是交给服务端模型或返回明确的模板外提示。

浏览器会优先请求同源 `/api/generate`。如果静态演示页没有 API，前端会回退到本地 `Talk2GraphEngine` 生成内置模板；模板无法识别的提示再交给服务端 LLM fallback。

## System Prompt 设计

固定注入到每轮 LLM 调用，核心约束：

1. 所有输出必须是完整的 TikZ 文档（含 `\documentclass` ... `\end{document}`）
2. 标注文字用中文
3. 几何图形的长度、角度按用户要求精确设定
4. 函数图像用 `pgfplots` 的 `\addplot`
5. 辅助线用 `dashed` + `gray`
6. 修改时只改被要求的部分，其余保持原样

## 多轮修改实现

第一轮：模板匹配优先；未匹配时 System Prompt + 用户描述 → TikZ 代码

后续轮次：

```
System Prompt
+ 上一轮的完整 TikZ 源码（注入到 user message 中）
+ 用户的新修改要求
→ LLM 在旧代码上做增量编辑 → 新 TikZ 代码
```

关键技术点：修改器看到的是完整的旧代码，不是"上次生成的图片"。内置修改器通过自动测试确认未涉及的坐标和绘图命令保持不变；LLM fallback 仍需后续加入更强的 diff 校验。

## 部署

- 推荐：Node 服务部署，前端访问同源 `/api/generate`
- 模型：通过服务端环境变量 `OPENAI_BASE_URL`、`OPENAI_API_KEY`、`OPENAI_MODEL` 配置
- 静态资源：Node 服务只白名单暴露首页、前端脚本和模板生成器，不暴露服务端代码、包配置或环境文件
- 静态演示：COS 可上传 `index.html`、`src/browser-app.js` 和 `src/graph-engine.js`，内置模板可在没有 `/api` 的情况下直接使用
- 待补：生产 Node 环境中的 node-tikzjax 渲染链路、字体资源和模型配置需要部署后复验
