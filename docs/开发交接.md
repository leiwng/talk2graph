## 产品边界

talk2graph 的 MVP 目标是让 K12 数学老师用中文自然语言生成可预览、可导出的数学图形，并支持基于已有图形继续修改。

必须优先保护的体验：

- 高频 K12 图形要确定性生成，不能靠不稳定模型输出碰运气。
- 图形比例、角度、坐标刻度和中文标注要可检查。
- 连续修改必须尽量保持未涉及部分不变。
- SVG 预览和 TikZ 导出都要保留。
- 浏览器端不得暴露模型密钥。

## 架构准则

核心原则：代码即状态，渲染是副作用。

- `src/graph-engine.js` 是内置模板和修改器的核心。
- `server.js` 只负责静态资源、`/healthz`、`/api/generate` 和模型 fallback 编排。
- `src/browser-app.js` 负责前端交互、示例按钮、历史状态、下载和本地模板 fallback。
- `src/llm-client.js` 只处理 OpenAI 兼容 Chat Completions 调用，不把密钥放到前端。
- `src/tikz-renderer.js` 是服务端 TikZ 到 SVG 的兜底渲染，不作为内置模板中文预览的首选路径。
- `scripts/verify-service.js` 是公网 Node 服务验收门禁。
- `scripts/evaluate-dataset.js` 是数据集覆盖门禁。

## 模板实现范式

新增图形时优先按现有模板模式实现：

1. 写提取函数，解析中文 prompt 中的参数。
2. 写几何计算函数，集中处理坐标、长度、角度、交点等数学关系。
3. 写 TikZ 模板函数，用于导出。
4. 写 SVG 模板函数，用于稳定中文预览。
5. 在 `tryTemplate` 中按“更具体优先”的顺序匹配，避免通用模板吞掉复杂请求。
6. 增加 `test/graph-engine.test.js` 覆盖精确坐标、关键标签、SVG 和 TikZ。
7. 如果来自 `docs/测试数据集.md`，同步保证 `npm run evaluate:dataset` 不退化。

不要用一个宽泛正则把未支持的复杂请求硬套进相似模板。宁可返回模板外提示，也不要生成语义错误图。

## 连续修改范式

连续修改必须基于当前 TikZ 源码或已有模板状态，目标是保持未涉及部分不变。

已有修改器覆盖：

- 去掉边长标注并添加斜边高。
- 直角标记外移。
- 添加辅助线、辅助线改虚线。
- 标出角 A/B/C。
- 修改函数坐标范围。
- 修改部分图形样式或多边形边数。

新增修改器时：

- 先写测试证明原坐标、原图元或原曲线仍存在。
- 修改行为要限制在匹配的图形类型上。
- 不要让直角三角形修改器误改等腰三角形、函数图或统计图。

## 部署准则

公网 Node 服务优先于 COS 静态页。

发布流程：

```bash
npm run package:service
```

上传 `dist/talk2graph-service.tar.gz` 到服务器 `/tmp/talk2graph-service.tar.gz` 后，在服务器执行：

```bash
tar -xOf /tmp/talk2graph-service.tar.gz talk2graph-service/deploy/install-cvm.sh \
  | sudo bash -s -- /tmp/talk2graph-service.tar.gz
```

公网验收：

```bash
TALK2GRAPH_SERVICE_URL=http://42.193.116.139:3000 npm run verify:service
```

验收必须看到 `/healthz` 的 `commit` 等于当前发布包 `build-info.json` 中的 commit。
